# Lab 1: Парсер логических формул

Проект реализует прямой `tree-first` разбор логических формул из инфиксной записи в дерево выражения без использования ОПЗ, postorder/postfix-представлений, shunting-yard и любых промежуточных линейных форматов такого типа.

## Возможности

- разбор формул напрямую в AST;
- поддержка операторов `!`, `/\`, `\/`, `->`, `~`;
- поддержка скобок и переменных вида `A`, `X1`, `VAR2`, `VAR_2`;
- поддержка логических констант `0` и `1`;
- структурная проверка, является ли формула СКНФ;
- загрузка формулы из текстового файла через консольное меню;
- понятные сообщения об ошибках;
- unit-тесты с высоким покрытием.

## Грамматика

Используется рекурсивный спуск с прямым построением дерева.

```text
formula       := equivalence EOF
equivalence   := implication ( "~" implication )*
implication   := disjunction ( "->" implication )?
disjunction   := conjunction ( "\/" conjunction )*
conjunction   := unary ( "/\" unary )*
unary         := "!" unary | primary
primary       := IDENTIFIER | CONSTANT | "(" equivalence ")"
IDENTIFIER    := LETTER ( LETTER | DIGIT | "_" )*
CONSTANT      := "0" | "1"
LETTER        := "A".."Z" | "a".."z"
DIGIT         := "0".."9"
```

### Приоритет операторов

От большего к меньшему:

1. `!`
2. `/\`
3. `\/`
4. `->`
5. `~`

Импликация разбирается как правоассоциативная операция: `A -> B -> C` интерпретируется как `A -> (B -> C)`.

## Ограничения и принципы

- парсер не использует ОПЗ, postfix, shunting-yard и похожие промежуточные представления;
- не используются внешние библиотеки для парсинга, построения дерева или CLI;
- дерево строится напрямую во время рекурсивного спуска;
- поддерживаются логические переменные и константы `0/1`; `True/False` не поддерживаются;
- составные выражения с бинарными операторами должны быть покрыты внешними скобками;
- отрицание допустимо перед переменной или подформулой при обычном парсинге, но для СКНФ отрицание над подформулой считается недопустимым.

## Как определяется СКНФ

В этом проекте используется структурная проверка по уже построенному дереву:

- вся формула должна быть конъюнкцией одной или нескольких клауз;
- клауза должна быть дизъюнкцией литералов;
- литерал — это либо переменная, либо отрицание переменной;
- отрицание вида `!(A \/ B)` или `!!A` не считается литералом;
- в каждой клаузе каждая переменная должна встречаться ровно один раз;
- все клаузы должны содержать один и тот же набор переменных;
- таблица истинности для такой проверки не строится: она нужна для получения СКНФ произвольной функции, а не для распознавания уже записанной СКНФ по структуре.

## Примеры корректных выражений

```text
A
!A
!0
!1
((A \/ !B) /\ (!A \/ B))
((!A \/ B \/ C) /\ (A \/ !B \/ !C))
(!(A /\ B) -> C)
(X1 /\ (VAR2 \/ !Z3))
(!(!(!0)) /\ 1)
```

## Примеры некорректных выражений

```text

A +
(A /\ B
A /\ \/ B
A B
!
A /\ B
!(!(!0 /\ 1) \/ (!!1 /\ !0)) ~ (0 \/ !1)
A)
```

## Пример запуска

Из каталога `lab1`:

```bash
python run.py
```

Пример сессии:

```text
Парсер логических формул.

1. Ввести формулу вручную
2. Загрузить формулу из файла
0. Выход
Выберите пункт: 1
Введите формулу: ((A \/ !B) /\ (!A \/ B))
СКНФ: да

1. Ввести формулу вручную
2. Загрузить формулу из файла
0. Выход
Выберите пункт: 2
Введите путь к файлу: formula.txt
Формула: !(A /\ B)
СКНФ: нет

1. Ввести формулу вручную
2. Загрузить формулу из файла
0. Выход
Выберите пункт: 0
Завершение работы.
```

Файл с формулой должен быть текстовым в кодировке UTF-8. Пример содержимого:

```text
((!A \/ B) /\ (A \/ !B))
```
